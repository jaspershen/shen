MetDNA-metID

Aug 12, 2019 6 min read R, Metabolomics

尽管现在除了二级谱图（MS2 spectra）匹配之外，已经有了各种各样的利用各种方式原理进行代谢物鉴定的方法，算法和工具，但是基于MS2谱图匹配的方法，仍然是最为可靠，为人接受，以及方便的方法。比如MetDNA，其前提条件就是数据一定是要尽可能完整的untargeted metabolomics数据。所以，前段时间，我把MetDNA中用于二级谱图匹配的算法单独拿出来，并进行了一定的优化，然后又新加了能够用于in-house代谢物库构建的功能，并将一些常见的public database下载下来，合并成了metID。

Introduction & Installation

metID 有两个核心功能：in-house database的构建和利用MS2 spectra进行代谢物鉴定。

很多实验室都有自己的in-house standards，并且大都采集了MS2数据，并且还有RT信息，但是很多实验室并没有将其构建成真正的in-hosue database，以及配套的软件来自动化利用其进行代谢物鉴定。metID因此提供了这样的功能，能够将实验室的标准品数据自动构建为适用于metID的database。根据MSI的规则，我们知道使用in-hosue database鉴定出的物质就是level 1。而很多我们熟知的database，如HMDB，MoNA和MassBank等，他们都提供了可下载的数据库，但是很多时候，在不同平台之间切换，费事费力，有的甚至不提供batch处理的功能，更是要命。因此，我把几个常见的public database下载下来，并构建成符合metID的数据库格式，并提供给大家使用。后面如果有新的数据库，可以再下载整理。

metID的源代码再github上托管，大家可以通过github下载。https://github.com/jaspershen/metID

Database construction

If you have in-house standards which have been acquired with MS2 spectra data, then you can construct the in-house MS2 spectra databases using metID package.

Data preparation

首先，请将标准品原始质谱数据使用 ProteoWizard转换为mzXM格式的数据。转换时参数设置可以参考下图。

Data organization

其次，需要将你的标准品信息整理成一个表格，然后输出为csv或者xlsx格式。标准品信息表格的格式可以参考我放在 demoData 包中的示例数据。

然后创建一个文件夹，并将你的mzXML格式数据(正离子数据放在’POS’文件夹中，负离子数据放在’NEG’文件夹中)和标准品信息表格（csv或者xlsx格式）放在里面。

Run databaseConstruction function

我们使用demoData包中的示例数据来进行说明，如何使用metID构建实验室自己的in-house database。首先安装该包。

devtools::install_github("jaspershen/demoData")
library(demoData)
library(metID)
path <- system.file("database_construction", package = "demoData")
#将示例数据拷贝到当前工作目录中
file.copy(from = path, to = ".", overwrite = TRUE, recursive = TRUE)
new.path <- file.path("./database_construction")

下面开始构建数据库。

test.database <- databaseConstruction(path = new.path, 
                                      version = "0.0.1",
                                      metabolite.info.name = "metabolite.info_RPLC.csv", 
                                      source = "Michael Snyder lab", 
                                      link = "http://snyderlab.stanford.edu/",
                                      creater = "Xiaotao Shen",
                                      email = "shenxt1990@163.com",
                                      rt = TRUE,
                                      mz.tol = 15,
                                      rt.tol = 30,
                                      threads = 3)

如果想要了解databaseConstruction各个参数的含义，可以使用?databaseConstruction来查看。

test.database是一个databaseClass对象，你可以直接打印，查看他的信息。

test.database

这样，就构建好了一个in-house database。

Metabolite identification

Identify metabolites based on MS2 library

MS1 data preparation

首先需要准备MS1 peak table。MS1 peak table必须包含的信息包括：“name” (peak name), “mz” (mass to charge ratio)和“rt” (retention time, unit is second)。MS1 peak table可以来自于各种不同的软件，如XCMS, MS-DIAL等。

MS2 data preparation

MS2原始数据可以来自于DDA或者DIA方法，然后使用ProteoWizard软件转换为msp, mgf或者mzXML格式的文件。

Data organization

将MS1一级数据，二级数据以及你要使用的database放置在某个文件夹中。

Run metIdentify function

下面我们使用demoData中的示例数据演示如何使用我们实验室的in-house database来进行代谢物鉴定。

library(metID)
path <- system.file("ms2_identification_demo_data1", package = "demoData")
#拷贝数据到当前工作目录中。
file.copy(from = path, to = ".", overwrite = TRUE, recursive = TRUE)
new.path <- file.path("./ms2_identification_demo_data1")

data("msDatabase_rplc0.0.1", package = "metID")
save(msDatabase_rplc0.0.1, file = file.path(new.path, "msDatabase_rplc0.0.1"))
ms2.data <- grep(pattern = "mgf", dir(new.path), value = TRUE)

然后使用metIdentify函数进行鉴定。

result <- metIdentify(ms1.data = "ms1.peak.table.csv", ##csv format
           ms2.data = ms2.data,##only msp and mgf and mz(X)ML are supported
           ms1.ms2.match.mz.tol = 25,
           ms1.ms2.match.rt.tol = 10,
           ms1.match.ppm = 25,
           ms2.match.tol = 0.4,
           fraction.weight = 0.3,
           dp.forward.weight = 0.6,
           dp.reverse.weight = 0.1,
           rt.match.tol = 60,
           polarity = "positive", 
           ce = "all",
           column = "rp",
           ms1.match.weight = 0.25,
           rt.match.weight = 0.25,
           ms2.match.weight = 0.5,
           path = new.path,
           total.score.tol = 0,
           candidate.num = 3,
           database = "msDatabase_rplc0.0.1",
           threads = 3)

metIdentify参数的意义可以使用?metIdentify查看。

生成的result是一个metIdentifyClass对象。

result

可以从result中提取你处理的参数设置：

parameters <- getParams(object = result)
tibble::as.tibble(parameters)

从中获取鉴定表格：

identification.table1 <- getIdentificationTable(object = result)

MS2 spectra match plot output

你也可以使用ms2plot函数来获得一个或者所有的MS2 specra match plot。

Output one MS2 spectra match plot.

##which peaks have identification
peak.name <- whichHasIden(object = result)
head(peak.name)
ms2.plot1 <- ms2plot(object = result, database = msDatabase_rplc0.0.1, 
                     which.peak = peak.name[1,1])
ms2.plot1

You can also output interactive MS2 spectra match plot by setting interaction.plot as TRUE.

##which peaks have identification
ms2.plot2 <- ms2plot(object = result, database = msDatabase_rplc0.0.1, 
                     which.peak = peak.name[1,1], interaction.plot = TRUE)
ms2.plot2

You can output all MS2 spectra match plots by setting which.peak as “all”.

ms2plot(object = result, database = msDatabase_rplc0.0.1, 
                     which.peak = "all", path = file.path(new.path, "inhouse"))

所有的MS2 spectra match会存放在“inhouse”文件夹中。
